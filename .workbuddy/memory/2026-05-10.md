   - 修复：使用数组索引 `status[0]`, `status[1]`, `status[4]` 访问返回值
   - 文件：`test/LiquidityManager.test.ts` line 187-189

2. **LiquidityManager.test.ts - Emergency Withdraw event 测试**
   - 问题：只 approve 了 AFC token，调用 `reserveLiquidity` 时缺少 USDT approve
   - 修复：添加 `await usdtToken.approve(liquidityManager.getAddress(), INITIAL_LIQUIDITY_USDT);`
   - 文件：`test/LiquidityManager.test.ts` line 268-269

3. **LiquidityManager.test.ts - Uniswap V3 factory address 测试**
   - 问题：测试期望非零地址，但 `UNISWAP_V3_FACTORY_BASE` 是零地址占位符
   - 修复：修改测试期望零地址，并添加注释说明
   - 文件：`test/LiquidityManager.test.ts` line 306-310

4. **Security.test.ts - should only allow owner to create vesting schedule**
   - 问题：`AFC_Token` 没有 `mint` 函数（所有代币在构造函数中一次性铸造）
   - 修复：改为测试 `createVestingSchedule`（只有 owner 可以调用）
   - 文件：`test/Security.test.ts` line 23-27

5. **Security.test.ts - should prevent unauthorized vesting creation**
   - 问题：同上，调用不存在的 `mint` 函数
   - 修复：改为测试 `createVestingSchedule`
   - 文件：`test/Security.test.ts` line 93-97

6. **Security.test.ts - should prevent unauthorized proposal creation**
   - 问题：`createProposal` 先检查代币余额，attacker 代币不足时回退信息是 "Insufficient tokens to propose" 而不是 "Ownable: caller is not the owner"
   - 修复：修改期望的回退信息
   - 文件：`test/Security.test.ts` line 110-114

### 技术发现

1. **ethers v6 多值返回处理**：
   - Solidity 函数返回多值时，ethers v6 返回数组（如 `status[0]`, `status[1]`）
   - 也可以使用命名属性（如果 ABI 中有结构体定义）

2. **AFC_Token 代币经济学**：
   - 所有代币在构造函数中一次性铸造（`MAX_SUPPLY = 10亿`）
   - 没有公开的 `mint` 函数（固定供应模式）
   - 代币分发通过 `transfer` 或 `batchTransfer` 完成

3. **LiquidityManager 设计**：
   - `getLiquidityStatus()` 返回流动性池中的总代币数量（包括预留的和合约持有的）
   - `reserveLiquidity` 需要同时 approve AFC 和 USDT 代币
   - `UNISWAP_V3_FACTORY_BASE` 当前是零地址占位符

## 赛博朋克 UI 增强工作

### 任务概述

为 net4.xyz 官方网站添加赛博朋克风格的视觉效果，提升未来科技感。

### 实施的三个效果

#### 1. ✅ 为 HeroSection 添加故障效果（Glitch Effect）

**文件**: `packages/frontend/src/components/Hero/HeroSection.tsx`

**修改内容**:
- 为标题中的"人类文明"文字添加了 `glitch-effect` CSS 类
- 添加了 `data-text="人类文明"` 属性（用于 CSS 伪元素显示故障文字）

**技术实现**:
- 使用 `::before` 和 `::after` 伪元素创建两层故障效果
- 使用 `clip-path` 分割文字上半部分和下半部分
- 使用 `glitchTop` 和 `glitchBottom` 动画移动伪元素
- 主元素使用 `glitch` 动画产生抖动效果

#### 2. ✅ 为 page.tsx 中的卡片添加玻璃拟态（Glassmorphism）

**文件**: `packages/frontend/src/app/page.tsx`

**修改内容**:
- 将 5 个卡片的 `bg-white/5 border border-white/10` 替换为 `glass-cyber` 类
- 为 Feature 卡片添加了 `glass-cyber-hover` 类（鼠标悬停时增强光晕）

**涉及的卡片**:
1. Feature 卡片（3个）- line 128
2. Whitepaper 内容卡片 - line 176
3. PoUE 统计卡片（3个）- line 190, 194, 198
4. Nodes 状态卡片 - line 217

**技术实现**:
- 使用 `backdrop-filter: blur(10px)` 实现毛玻璃效果
- 半透明背景 `rgba(255, 255, 255, 0.03)`
- 紫色边框 `rgba(176, 38, 255, 0.2)`
- 多层 `box-shadow` 实现内发光和外发光

#### 3. ✅ 为按钮添加霓虹光晕（Neon Glow）

**文件**: 
- `packages/frontend/src/components/Hero/HeroSection.tsx`
- `packages/frontend/src/app/page.tsx`

**修改内容**:
- "Ignite Spark" 按钮 → 添加 `neon-glow-pink`
- "了解更多" 按钮 → 添加 `neon-glow-purple`
- "向 AI 询问 PoUE 原理" 按钮 → 添加 `neon-glow-purple`
- "搜索" 按钮 → 添加 `neon-glow-cyan`

**技术实现**:
- 使用多层 `box-shadow` 创建立体霓虹效果
- 第一层：5px 模糊（紧贴元素）
- 第二层：10px 模糊（中等光晕）
- 第三层：15-20px 模糊（远距光晕）
- 配合 `neonFlicker` 动画实现闪烁效果

### 其他赛博朋克样式（已定义但未使用）

- `.cyber-grid-bg` - 赛博朋克网格背景（静态）
- `.cyber-grid-bg-animated` - 赛博朋克网格背景（动态流动）
- `.holographic-gradient` - 全息渐变背景
- `.data-stream` - 数据流动画

### 技术要点

1. **故障效果依赖 `data-text` 属性**:
   - CSS 的 `content: attr(data-text)` 需要元素有 `data-text` 属性
   - 伪元素会显示 `data-text` 的内容，产生重影效果

2. **玻璃拟态需要半透明背景**:
   - 不能使用 `bg-white/5`（Tailwind 不生效）
   - 必须使用 CSS 自定义属性或直接写 `rgba()` 颜色

3. **霓虹光晕的性能考虑**:
   - 多层 `box-shadow` 可能影响性能
   - 建议在悬停时启用，或限制使用数量
